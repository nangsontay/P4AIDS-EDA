Chapter 3 Class & Scale Imbalance

Objective: Dissecting the Dual Imbalance phenomenon — frequency AND area — that makes ADE20K uniquely challenging.

3.1 — Extreme Long-tail Distribution

3,490

43,470
"Window" — Top Class Count

The top ~10-15 classes account for over 40% of all instances, with "window" leading at 43,470 instances. Meanwhile, 3,058 classes (87.6%) appear fewer than 100 times — a textbook extreme long-tail distribution.

Critical

Must use mIoU (not Pixel Accuracy) as the evaluation metric — Pixel Accuracy will be dominated by frequent classes and mask the model's failure on 87.6% of all categories.

Loss Strategy

Use Focal Loss or Weighted Cross-Entropy to up-weight rare classes and prevent the model from ignoring the long tail.

The Paradox

"Window" appears most frequently (43,470 instances) but "Wall" dominates total pixel coverage. This dual imbalance means optimizing for one metric can actively harm the other — the loss function must balance both frequency and area.

3.3 — Pixel-Level Scale Imbalance

Large objects (diff, court, ruins, sea water) occupy 20%-100% of image area, while micro objects (hand, eye, mouth, knob, hinge) consistently occupy <1% of area.

Architecture Imperative

CNN downsampling will erase micro-objects entirely — FPN/BiFPN/U-Net skip connections are mandatory to recover fine-grained spatial features from early layers.

Chapter 4 Contextual Relationships & Spatial Logic

Objective: Giving the model "spatial awareness" via co-occurrence, location, and occlusion analysis.

4.1 — Co-occurrence Heatmap

The heatmap shows conditional probability P(Column|Row) — the probability of one object appearing given another is present.

Key Co-occurrence Clusters

  • Indoor Cluster: wall ↔ floor ↔ ceiling — these three almost always co-occur
  • Outdoor Cluster: car ↔ road ↔ building — strong mutual dependency
  • Mutual Exclusion: bed-road never co-occur — enforcing scene-type separation
Architecture Implications

Use Transformer Self-Attention (SegFormer, Mask2Former) to learn these co-occurrence patterns naturally, or CNN + CRF (Conditional Random Field) post-processing to enforce spatial consistency.

4.2 — Spatial Prior / Location Heatmap

Y=0.0-0.3
Sky & Ceiling (Top)
Y=0.6-1.0
Floor & Road (Bottom)

Objects exhibit strong spatial gravity priors — sky and ceiling concentrate at the top, while floor and road dominate the bottom. This is a fundamental property of natural images.

Critical Rule

ABSOLUTELY NO Vertical Flip augmentation — it would place sky at the bottom and floor at the top, destroying the spatial prior the model needs to learn.

Architecture Solution

CNNs have Translation Invariance and cannot encode position natively. Add CoordConv layers (for CNN) or keep Positional Encoding (for ViT) to inject location awareness.

Chapter 5 Preprocessing Pipeline & Baseline Model

5.1 — Data Augmentation Strategy

Pipeline Steps (Evidence-Driven)

1

Longest-edge Resize (512) + Zero-Padding

From Insight 2.1 — Preserves aspect ratio, no geometric distortion. All images → uniform 512×512 tensors.

2

RandomHorizontalFlip only — NO Vertical Flip

From Insight 4.2 — Preserves spatial gravity prior (sky=top, floor=bottom).

3

Random Crop + Random Erasing / CutOut

From Insights 2.2 & 4.3 — Simulates dense scene cluttering and real-world occlusion patterns.

4

Color Jitter (brightness/contrast/saturation)

From Insight 1.3 — Handles diverse indoor/outdoor lighting conditions.

5

Copy-Paste Augmentation for rare classes

From Insight 3.1 — Directly balances the extreme long-tail distribution by pasting rare-class masks into scenes.

Library Choice

Use the Albumentations library (not torchvision.transforms) — it supports simultaneous image+mask transforms, is significantly faster, and has richer augmentation options for segmentation tasks.

5.2 — Baseline Model Architecture & Loss Function

Phase 1: U-Net + ResNet-50

  • Skip connections solve micro-object feature loss
  • Encoder-decoder architecture recovers spatial resolution
  • Fast to train, good baseline benchmark

Phase 2: SegFormer

  • Transformer SOTA for semantic segmentation
  • Global receptive field via Self-Attention
  • Learns co-occurrence patterns naturally (Ch.4)

Loss Function

L_total = α · L_Focal + β · L_Dice    (α=0.5, β=0.5)

• Focal Loss → heavy penalty on rare/hard classes
• Dice Loss → optimizes IoU directly, scale-invariant
Evaluation Metric

mIoU only (not Pixel Accuracy) — ensures fair evaluation across all 3,490 classes. Pixel Accuracy would be misleadingly high due to dominant background classes.

Summary Key Insights from ADE20K EDA

  • Zero Data Corruption
    All 25,574 image-mask pairs pass integrity checks — 0 non-RGB images, 0 dimension mismatches. The dataset is production-ready with no preprocessing exceptions needed.
  • Extreme Long-tail: 87.6% Rare Classes
    3,058 out of 3,490 classes appear fewer than 100 times. Standard cross-entropy will ignore the tail entirely — Focal Loss + Copy-Paste Augmentation are mandatory.
  • 59.1% Small Objects Dominate
    Over half of all objects occupy <2% of image area. Multi-scale feature extraction (FPN/BiFPN) and skip connections are non-negotiable architectural requirements.
  • Spatial Gravity Priors Are Real
    Sky/ceiling concentrate at Y=0.0-0.3, floor/road at Y=0.6-1.0. Vertical flip augmentation would destroy this prior — only horizontal flip is permitted.
  • Dual Loss + Transformer = Optimal Baseline
    L_total = 0.5·Focal + 0.5·Dice handles both frequency and area imbalance. SegFormer's self-attention naturally captures co-occurrence patterns (wall↔floor↔ceiling).
